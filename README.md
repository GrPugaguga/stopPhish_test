# StopPhish Notes

Выполнено тестовое задание: https://drive.google.com/file/d/1fqOfqSzb2vlWznYg46fjrN_hgzH-2mvy/view

Техническая документация и инструкция по запуску в Assignment.md

На выполнение ушло порядка 8 часов, все доп задачи выполнены.

Из неочевидного нужно было разобраться с moleculer (в особенности с gateway и способом валидации DTO), в конце базово разобрался с k8s чтобы выполнить все задания

Финальный стек:
Node.js, TypeScript, Vue3, Vite, Tailwind, Pinia, Vue Router, Axios,
Moleculer, TypeORM, PostgreSQL, Zod (валидация), bcryptjs + jsonwebtoken (для работы с JWT), NATS в качестве транспортера

В начале работы собрал базовый CI в .github/workflows/ci.yml с проверкой линтера, форматера и тестов (добавил один тест пустышку)

Далее собрал базовый docker-compose.yml с конфигами для NATS и PostgreSQL.

Согласно заданию backend разделил на users сервис (отвечает за авторизацию и аутентификацию пользователей) и notes (отвечает за категории и заметки)
Также разделил и две PostgreSQL базы данных users-db и notes-db

Для удобной работы отдельно создал shared папку.

Внутри shared/config/env.ts расписал zod валидацию и экспорт всех переменных из env
Внутри shared/schemas расписал все типы DTO и Response для использования в сервисах и на фронте
Отдельно написал хелпер shared/schemas/validator.ts для валидации всех входящих данных

В сервисе notes также создал сущность категорий, связь к заметкам как один ко многим, у каждого пользователя свои категории. Заметка может быть создана как с привязкой к определённой categoryId так и без неё
Полнотекстовый поиск проходит по имени заметок и осуществлён через ILIKE запросы к БД
Все CRUD операции с категориями и заметками защищены через JWT token

Изменения структуры базы данных записаны внутри папок migrations в сервисах сущностей

Gateway сервис служит только для проксирования запросов, защищённые ручки сначала проходят через middleware, который проверяет JWT в users сервисе. Добавлен обработчик onError для валидации ошибок, созданы типы shared/errors для выбрасывания кастомизированных ошибок

frontend сервис на две страницы — логин и основная страница с заметками.

Внутренняя логика связи с backend и сохранения в памяти сервиса расписана в frontend/src/stores, выделен frontend/src/ui-kit с базовыми элементами и frontend/src/components с собранными компонентами.
За роутинг отвечает frontend/src/router

Для группировки заметок добавлен базовый drag&drop, также менять категорию можно через попап редактирования

Для frontend и backend сервисов собраны докерфайлы и все сервисы добавлены в docker-compose.yml для удобных тестов
